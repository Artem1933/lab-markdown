## Відповіді на контрольні запитання

1.  **Що таке Markdown?**
    Це спрощена мова розмітки тексту, призначена для написання максимально читабельного тексту, який можна легко перетворити на HTML (для веб-сторінок), PDF або DOCX.

2.  **Основні елементи форматування:**
    Заголовки (`#`), жирний (`**`) та курсивний (`*`) текст, списки (марковані `-`, нумеровані `1.`), цитати (`>`), код (`` ` ``) та посилання.

3.  **Вставка формул:**
    Використовується синтаксис LaTeX.
    * Інлайн: `$E=mc^2$`
    * Блочна: `$$x=y$$`

4.  **Що таке Mermaid?**
    Це інструмент для створення діаграм та графіків за допомогою коду. Однак при простій конвертації у PDF він часто залишається текстом, тому для гарантованого відображення використовують ASCII-схеми або картинки.

5.  **Конвертація Pandoc:**
    Виконується командою в терміналі: `pandoc file.md -o file.pdf`. Для PDF часто потрібен додатковий рушій (xelatex).

6.  **Структура проєкту:**
    * `README.md` - опис проєкту.
    * `docs/` - папка зі звітами.
    * `dist/` - папка з готовими файлами (PDF, HTML).

7.  **GitHub Pages:**
    Налаштовується в Settings -> Pages. Дозволяє опублікувати HTML-файл з папки `/docs` як справжній сайт.

8.  **Переваги Markdown:**
    Універсальність, малий розмір файлів, зручність контролю версій (Git), фокус на змісті, а не на оформленні.

9.  **Проблеми з PDF:**
    Основна проблема - відображення кирилиці (вирішується шрифтами) та рендеринг складних графіків (Mermaid).

10. **Вимоги до звіту:**
    Звіт має містити формули, таблиці, діаграму і бути збереженим у форматах HTML, DOCX, PDF.
